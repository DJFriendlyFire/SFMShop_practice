# Архитектура асинхронной обработки заказов для проекта SFMShop

## Задачи для очереди

### 1. Отправка email-уведомлений
- **Задача**: send_email
- **Триггер**: Создание заказа
- **Данные**: order_id, user_email
- **Приоритет**: Низкий (не критично для ответа API)

### 2. Обновление склада
- **Задача**: update_stock
- **Триггер**: Создание заказа
- **Данные**: order_id, items (список товаров)
- **Приоритет**: Высокий (важно для бизнес-логики)

### 3. Генерация отчетов
- **Задача**: generate_report
- **Триггер**: Создание заказа
- **Данные**: order_id
- **Приоритет**: Низкий (можно обработать позже)

### 4. Отправка push-уведомлений
- **Задача**: send_notification
- **Триггер**: Изменение статуса заказа
- **Данные**: order_id, status
- **Приоритет**: Средний

## Надежность

### Гарантии доставки
- **Persistent messages**: Сообщения сохраняются на диск
- **Acknowledgment**: Consumer подтверждает обработку сообщения
- **Если Consumer упал**: Сообщение возвращается в очередь для повторной обработки

### Retry-логика
- **Количество попыток**: 3 попытки при ошибке
- **Задержка между попытками**: Экспоненциальная (1 сек, 2 сек, 4 сек)
- **После исчерпания попыток**: Сообщение отправляется в очередь ошибок для ручной обработки

### Обработка ошибок
- **Логирование**: Все ошибки логируются
- **Мониторинг**: Отслеживание количества ошибок
- **Алерты**: Уведомления при большом количестве ошибок

## Масштабирование

### Горизонтальное масштабирование Consumer
- **Несколько воркеров**: Запуск нескольких экземпляров Consumer
- **Балансировка нагрузки**: RabbitMQ автоматически распределяет сообщения между воркерами
- **Автоматическое масштабирование**: Добавление воркеров при росте нагрузки

### Мониторинг
- **RabbitMQ Management Interface**: Мониторинг очередей, сообщений, воркеров
- **Метрики**: Количество сообщений в очереди, скорость обработки, ошибки
- **Алерты**: Уведомления при проблемах (переполнение очереди, ошибки)

## Интеграция с проектом

### Producer (src/api/main.py)
- Endpoint POST /orders отправляет задачи в очередь
- Быстрый ответ пользователю (не ждет обработки задач)

### Consumer (src/services/queue_consumer.py)
- Получает задачи из очереди
- Обрабатывает задачи в фоне
- Интегрируется с существующими сервисами (email, stock, reports)
